Criar branches para funcionalidades(Atualmente existe apenas main e develop no repo remoto)
Já que você tem main e develop, para cada nova “parte” crie uma branch de feature, por exemplo:

git checkout develop  
git checkout -b feature/cadastro  


Trabalhe nessa branch até que o cadastro funcione. Depois merge para develop, etc. Isso segue um padrão de branching profissional (como o modelo GitFlow — onde main é versão “pronta”, develop para integração, e features para desenvolvimento).
